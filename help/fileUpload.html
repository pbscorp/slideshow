Select “Record Audio: ” to start recording a script that will be used a slide, then upload the file on the recorder, the recorder will upload the file with a temporary name and refresh the media list.  Rename the slide to match the name to the corresponding slide.

 

Select media files from your local storage “Select file:  📁” to be staged for uploading,

then select “Upload Files: ”  to load them into your workspace.
Once loaded, the media list will be refreshed with the added files.’

 

An image file named ‘default’ will be used as the background for the slideshow.

When the slideshow is presented, the files with base names that match the names on the slide text will be shown and/or heard when the matching slide name is processed in the slideshow. The files extension will determine the media type.


File names that do not match slide text names will be ignored and are highlighted in red text color these can be renamed.

 

Note:

Browser support for media types (audio/video formats and codecs) varies significantly due to licensing restrictions, development priorities, and performance considerations. 

 

The most common file extensions for browsers and websites, which I have found high compatibility, for standard image and media: 

Images: .jpg, .jpeg, and .png

Audio/Video: .mp3, .mp4, .m4v and .mov
